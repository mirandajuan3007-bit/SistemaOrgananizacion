# RF_08 — Seguimiento de comunicaciones mediante folio

## por qué existe este requisito en este sistema

Uno de los problemas más operativos que describiste en sesiones anteriores fue que la conversación entre quien revisa y quien responde se pierde en Gmail o Outlook. El correo seguirá siendo el canal de comunicación, pero no puede seguir siendo el canal de control.

La solución que se propuso para ese problema fue un esquema de seguimiento por folio. Este requisito funcional formaliza justamente esa necesidad: convertir el hilo de mensajes en parte del control administrativo del contrato o expediente.

---

## descripción del requisito

El sistema debe permitir asociar comunicaciones relevantes a un contrato, expediente o trámite mediante un folio identificador, para conservar seguimiento operativo sin depender de buscar manualmente en el correo.

El sistema debe permitir:

- generar o reutilizar un folio de seguimiento para un contrato o trámite
- vincular mensajes enviados y respuestas recibidas a ese folio
- visualizar el historial de comunicación asociado
- detectar periodos de inactividad o falta de respuesta
- reflejar la última interacción y el responsable pendiente de responder

---

## actores involucrados

- secretaría
- revisor interno
- participante o emisor externo, de forma indirecta por correo

---

## precondiciones

- debe existir un contrato, expediente o trámite con folio activo
- el sistema debe tener integración habilitada con el canal de correo o con el mecanismo de captura de comunicaciones

---

## flujo principal

1. El sistema genera o asigna un folio al trámite.
2. La secretaría envía un correo relacionado usando ese folio en asunto o referencia operativa.
3. El sistema registra el envío como evento de comunicación.
4. Cuando llega una respuesta relacionada, el sistema la asocia al mismo folio.
5. El expediente o contrato muestra el historial de interacciones y su estado de respuesta.

---

## flujos alternos y excepciones

### respuesta sin folio reconocible

1. Llega un correo sin el formato esperado.
2. El sistema no puede asociarlo automáticamente.
3. El usuario puede vincularlo manualmente al expediente correcto.

### múltiples mensajes en un mismo hilo

1. Se intercambian varias respuestas.
2. El sistema conserva el orden cronológico y marca quién respondió por última vez.

### comunicación cerrada

1. El asunto ya fue atendido.
2. El usuario marca el hilo como cerrado.
3. El sistema deja de considerarlo pendiente para alertas operativas.

---

## reglas de negocio asociadas

- toda comunicación operativa relevante debe poder vincularse a un folio
- el folio debe identificar unívocamente el trámite dentro de la unidad
- el sistema debe conservar la última fecha de interacción y el último emisor
- una falta de respuesta debe poder convertirse en alerta operativa

---

## cómo se mide este requisito en el sistema

El requisito se considera cumplido si:

- puede conocerse desde el expediente o contrato cuál fue la última respuesta y cuándo ocurrió
- el sistema puede mostrar hilos con seguimiento asociado a un folio
- la operación ya no depende de buscar manualmente en la bandeja completa de correo para saber el estado de un intercambio

---

## cómo se traduce en el sistema

### en el frontend

- existe una sección de comunicaciones dentro del contrato o expediente
- se muestra línea de tiempo de correos o eventos de comunicación
- se muestra indicador de días sin respuesta

### en el backend

- existe lógica de asociación por folio
- existe registro de eventos de comunicación entrante y saliente
- existe cálculo del estado de respuesta del hilo

### en la base de datos

- existe una tabla `comunicaciones` o equivalente
- cada comunicación se relaciona con `contrato` o `expediente`
- existe campo de folio, fecha, dirección del mensaje y estado de seguimiento

---

## cómo se evidencia que se cumple

- desde el contrato puede verse el historial de correos relacionados
- el sistema muestra quién fue la última persona en responder
- puede detectarse si han pasado 24, 48 o 72 horas sin respuesta
- el folio permite ubicar rápidamente la conversación correcta

---

## cómo se representa en diseño

### en casos de uso

Debe existir el caso de uso "dar seguimiento a comunicación por folio".

### en el modelo de datos

La entidad `Comunicación` debe asociarse al contrato o expediente y conservar referencia temporal.

### en diagramas de secuencia

Debe modelarse el envío, recepción y asociación automática o manual del mensaje al folio.
