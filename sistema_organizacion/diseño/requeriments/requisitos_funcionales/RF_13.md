# RF_13 — Asistente de IA contextual para consulta y apoyo operativo

## por qué existe este requisito en este sistema

Ya se definió en el análisis de arquitectura que el sistema no solo organizará datos, sino que también tendrá un asistente de IA con contexto real del sistema. Ese asistente no es un adorno. Tiene sentido porque la operación administrativa repite preguntas y tareas de alto costo cognitivo: qué falta, en qué estado está algo, qué pasó antes, qué debería hacerse después.

Este requisito es funcional porque el usuario interactúa con una capacidad concreta del sistema: preguntar, obtener respuesta contextual y, en ciertos casos, recibir ayuda para redactar o resumir información.

---

## descripción del requisito

El sistema debe ofrecer un asistente de IA contextual capaz de consultar información del sistema y apoyar tareas operativas autorizadas.

El asistente debe permitir:

- responder preguntas sobre estado de contratos, expedientes y proyectos
- resumir historial de un expediente o contrato
- indicar qué documentos faltan a un participante o proyecto
- buscar información dentro de documentos relacionados
- ayudar a redactar correos o respuestas con contexto del expediente
- proponer el siguiente paso operativo sin ejecutar acciones no autorizadas

---

## actores involucrados

- secretaría
- director
- administrador del sistema

---

## precondiciones

- el usuario debe haber iniciado sesión y tener permiso para usar el asistente
- el asistente debe operar solo sobre información a la que el usuario tenga acceso
- el sistema debe contar con datos y documentos indexados para consulta semántica cuando aplique

---

## flujo principal

1. El usuario abre el panel del asistente.
2. Escribe una pregunta en lenguaje natural.
3. El asistente interpreta la intención.
4. Consulta las herramientas internas necesarias del sistema.
5. Recupera datos o fragmentos documentales relevantes.
6. Construye y presenta una respuesta en lenguaje natural.
7. Si aplica, ofrece acciones sugeridas o borradores de texto.

---

## flujos alternos y excepciones

### consulta sin contexto suficiente

1. El usuario hace una pregunta ambigua.
2. El asistente solicita una aclaración concreta antes de responder.

### consulta sobre información no autorizada

1. El usuario pide datos a los que no tiene acceso.
2. El asistente no los expone y responde respetando permisos.

### respuesta basada en documentos

1. El usuario hace una pregunta sobre el contenido de un contrato o archivo.
2. El asistente busca en los documentos indexados.
3. Responde con base en el contenido encontrado y cita el origen interno si así se diseña.

---

## reglas de negocio asociadas

- el asistente solo puede consultar información permitida para el usuario autenticado
- el asistente no debe inventar estados ni datos que no existan en el sistema
- las acciones propuestas por el asistente no sustituyen reglas del flujo operativo formal
- si el asistente redacta un correo o resumen, el usuario debe poder revisarlo antes de usarlo

---

## cómo se mide este requisito en el sistema

El requisito se considera cumplido si:

- el usuario puede obtener respuestas útiles sobre el estado real del sistema sin navegar por múltiples pantallas
- el asistente identifica correctamente faltantes, estados y resúmenes en la mayoría de los casos esperados
- el asistente respeta permisos y no expone datos fuera del alcance del usuario
- el asistente reduce tiempo de consulta y trabajo repetitivo en tareas de redacción o resumen

---

## cómo se traduce en el sistema

### en el frontend

- existe un panel o componente de chat integrado a la aplicación
- el usuario puede hacer preguntas y recibir respuestas con streaming
- las respuestas pueden incluir enlaces o accesos al expediente, contrato o documento relacionado

### en el backend

- existe un módulo de IA con tools internas para consultar expedientes, contratos, documentos y pendientes
- el backend controla permisos antes de responder
- existe recuperación de contexto estructurado y semántico según el tipo de consulta

### en la base de datos y almacenamiento

- los datos estructurados se consultan desde PostgreSQL
- los documentos pueden indexarse para búsqueda semántica sobre pgvector o mecanismo equivalente

---

## cómo se evidencia que se cumple

- la secretaría puede preguntar "qué documentos le faltan a tal participante" y recibir la respuesta correcta
- el director puede pedir un resumen del estado de un proyecto y obtenerlo desde el sistema
- el asistente puede redactar un borrador de correo con contexto del expediente sin obligar al usuario a escribir desde cero
- si el usuario pregunta por datos fuera de su alcance, el asistente no los revela

---

## cómo se representa en diseño

### en casos de uso

Debe existir el caso de uso "consultar al asistente de IA".

### en la arquitectura

Debe representarse el módulo de IA conectado a datos estructurados, documentos y controles de permisos.

### en wireframes

Debe existir diseño del panel de chat y de la forma en que las respuestas enlazan al contexto operativo del sistema.
